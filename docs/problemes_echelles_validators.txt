PROBLÈMES D'ÉCHELLES TROUVÉS DANS LES VALIDATORS
=================================================

J'ai trouvé les problèmes d'échelle 0-100 vs 0-1 dans les validators !

1. RSI_Regime_Validator - PROBLÈME AVEC momentum_score
========================================================

Dans analyzer_data : momentum_score est 0-100
Dans le validator RSI_Regime_Validator :

Ligne 86 : if momentum_score > 30.0    # Traite comme 0-100 ✓
Ligne 115: if momentum_score < -30.0   # ERREUR ! momentum_score n'est jamais négatif
Ligne 196: if momentum_score > 50.0    # Traite comme 0-100 ✓
Ligne 198: if momentum_score < -50.0   # ERREUR ! momentum_score n'est jamais négatif

PROBLÈME : Le validator teste des valeurs négatives pour momentum_score, 
mais dans analyzer_data, momentum_score est toujours entre 0 et 100.

2. Volume_Quality_Score_Validator - CONVERSION MANUELLE
========================================================

Ligne 234: quality_normalized = volume_quality_score / 100.0

Le validator divise par 100 pour convertir de 0-100 vers 0-1.
C'est correct MAIS cela montre qu'il y a confusion sur les échelles.

3. ATR_Volatility_Validator - UTILISE atr_percentile
=====================================================

atr_percentile est bien 0-100 dans analyzer_data
Le validator l'utilise correctement comme 0-100

4. PROBLÈME GÉNÉRAL - INCOHÉRENCE DES ÉCHELLES
===============================================

Les validators doivent gérer des indicateurs avec différentes échelles :
- RSI, ADX, Stochastic : 0-100 ✓
- bb_position, natr : 0-1 ✓
- momentum_score : 0-100 (mais testé comme s'il pouvait être négatif)
- volume_quality_score : 0-100 (mais converti en 0-1 dans le validator)

5. POURQUOI LE SCORE FINAL EST 0-1
==================================

Malgré ces incohérences internes, chaque validator retourne un score entre 0-1 :
- validate_signal() → boolean
- get_validation_score() → float entre 0 et 1

C'est dans le CALCUL INTERNE du score que les validators font parfois
des erreurs de conversion ou de test.

RECOMMANDATIONS
===============

1. CORRIGER RSI_Regime_Validator :
   - Remplacer les tests < -30 et < -50 par < 30 et < 50
   - Ou utiliser une autre logique pour détecter le momentum bearish

2. UNIFORMISER la documentation :
   - Chaque validator devrait documenter l'échelle attendue
   - Ex: # momentum_score: 0-100 (jamais négatif)

3. CRÉER des helpers dans base_validator :
   - _normalize_percentage(value) : convertit 0-100 vers 0-1
   - _is_bullish_momentum(score) : teste si > 50
   - _is_bearish_momentum(score) : teste si < 50

Le problème principal est dans RSI_Regime_Validator qui teste 
des valeurs négatives impossibles pour momentum_score.
